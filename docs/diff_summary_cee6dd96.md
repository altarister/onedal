# 🚀 중간 점검: 최근 업데이트 변경점 요약
**(기준 커밋: `cee6dd96aa8dc251d5c9ef9e115b96bf5bb49eb9` ~ 현재)**

최근 진행된 개발은 단순히 UI를 고치는 것을 넘어, **"운행 상태의 영속성 확보"**와 **"다이나믹 필터의 자동화"**라는 큰 아키텍처 변화를 이루어냈습니다. 주요 변경 사항을 4가지 핵심 영역으로 나누어 정리했습니다.

---

## 1. ⚙️ 아키텍처: 단일 진입점(SSOT) 및 상태 머신 도입
가장 거대한 변화는 기사님의 필터 상태(`activeFilter`)를 관리하는 방식이 중앙 집중화된 것입니다.

*   **`filterManager.ts` 신설 (`applyFilter`)**: 이전에는 여러 곳에서 파편화되어 필터를 업데이트했지만, 이제는 오직 `applyFilter` 함수를 통해서만 상태가 바뀝니다. 이 함수는 **메모리(Session) + DB 영구 저장 + 웹소켓 전파**를 한 번에 원자적으로 처리합니다.
*   **상태 머신 (`loadState`) 도입**: 기사님의 현재 업무 진행도를 4단계(`EMPTY` 대기 -> `LOADING` 적재중 -> `DRIVING` 운행중 -> `ARRIVED` 도착)로 정의하고 SQLite DB에 영구 저장하도록 스키마를 마이그레이션했습니다.
*   **3단계 다이나믹 회랑(Corridor) 축소 로직**:
    *   **첫 짐 (EMPTY → LOADING)**: 주변 10km 반경까지 회랑을 넓게 잡아 합짐 탐색
    *   **출발 (LOADING → DRIVING)**: 회랑을 0km로 축소하여 오직 "가는 길"에 있는 콜만 스나이핑
    *   **실시간 Trim (`trimCorridorByProgress`)**: GPS 이동에 따라 이미 지나온 지역의 키워드는 필터에서 자동으로 삭제하여 뒤로 돌아가는 똥콜을 원천 차단

## 2. 🗺️ UI/UX: 지역 미리보기 (Regional Preview)
기사님이 목적지 시/도를 타이핑할 때, 어떤 읍/면/동이 타겟팅되는지 눈으로 먼저 확인할 수 있게 되었습니다.

*   **`geoResolver.ts` 업데이트**: 단순히 읍/면/동을 나열하는 대신, 상위 '구' 단위로 이쁘게 그룹핑(`getGroupedRegionsByCity`)하는 알고리즘을 추가했습니다.
*   **실시간 디바운스 API**: `OrderFilterModal.tsx`에서 시/도 이름을 타이핑하면 0.4초 대기 후 백엔드(`/api/settings/preview-regions`)를 찔러, 변경 예정인 타겟팅 지역들을 아코디언 메뉴로 미리 보여줍니다.

## 3. 🏠 신규 기능: 자동 귀가콜 (Home Return)
저녁 퇴근 시 빈 차로 돌아오는 손실을 막기 위한 전용 모드가 신설되었습니다.

*   **집 주소 저장**: `SettingsModal.tsx` 설정 창에 '집 주소'를 입력받고, 카카오 지오코딩을 통해 `home_x`, `home_y` 좌표로 변환하여 DB에 저장합니다.
*   **가상 오더 생성**: 대기 화면(Empty State)에 생긴 **[🏠 귀가콜 시작]** 버튼을 누르면, 현재 위치에서 집까지의 가상 오더를 서버가 즉시 생성하고 카카오 경로를 땁니다. 이후 자동으로 `LOADING` 상태에 돌입해 집으로 가는 길의 콜을 사냥합니다.

## 4. 📍 GPS 시뮬레이터 및 텔레메트리 분리
목업 테스트와 실제 GPS 추적 로직을 완전히 분리하여 안정성을 높였습니다.

*   **`useMockGpsSimulator.ts` 독립**: 이전처럼 컴포넌트 렌더링에 휘둘려 제자리걸음하던 버그를 잡고, 순수하게 좌표 배열만 받아서 15배속으로 주행하는 독립 훅(Hook)으로 빼냈습니다.
*   **도착 자동 감지**: `geoService.ts`에 하버사인(Haversine) 거리 계산 공식이 추가되어, 마스터 GPS가 최종 목적지 반경 500m 이내로 진입하면 시스템이 스스로 `ARRIVED` 상태로 전환합니다.
