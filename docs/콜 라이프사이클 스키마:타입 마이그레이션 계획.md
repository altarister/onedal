# 1DAL 콜 라이프사이클 스키마/타입 마이그레이션 계획

## 🔄 콜 라이프사이클 최종 정의 (V3)

```text
[1단계: 패널티 없는 목록 탐색]
ORDER_PRE_SECURED (가확정 전 검토) 
  - 인성 리스트 겉표면만 읽은 상태. 버려도 패널티 0%.
      ↓
[2단계: 패널티 위험 구간 진입]
ORDER_SECURED_EVALUATING (가확정 후 상세 분석)
  - 인성 상세 화면에 진입하여 배차를 "가확정(Secured)" 받은 상태.
  - 여기서 취소하면 패널티(혹은 30초 룰)가 적용됨!
  - 💡 기사님이 "수동"으로 직접 콜을 잡은 경우, 1단계를 건너뛰고
    바로 이 상태로 진입 → 수동/자동 콜 여부를 즉각 식별 가능.
      ↓
[3단계: 관제탑 결재]
ORDER_AWAITING_DECISION (관제탑 결재 대기 - 데스밸리)
      ↓
      ├── IF (KEEP 배차 수락)
      │       ↓
      │   ORDER_CONFIRMED (배차 확정 - 내 퀵)
      │       ↓
      │       ├── IF (정상 배송 완료) → ORDER_COMPLETED (운행 완료)
      │       └── ELSE (확정 후 내가 포기) → ORDER_RELEASED (배차 방출)
      │
      └── ELSE (CANCEL 배차 거절)
              ↓
              ORDER_CANCELED (내가 능동 거절)
              ORDER_FORCE_CANCELED (사무실/화주가 수동태로 취소시킴)
```

### 상태별 의미 요약

| 상태값 | 패널티 | 주체 | 설명 |
|:---|:---|:---|:---|
| `ORDER_PRE_SECURED` | ❌ 없음 | 시스템 | 리스트에서 겉표면 스캔 중 |
| `ORDER_SECURED_EVALUATING` | ⚠️ 있음 | 시스템/기사 | 상세화면 진입, 카카오 연산 중 |
| `ORDER_AWAITING_DECISION` | ⚠️ 있음 | 관제탑 | 관제탑 결재 대기 (데스밸리) |
| `ORDER_CONFIRMED` | ✅ 내 콜 | 관제탑 | KEEP 승인, 배차 확정 |
| `ORDER_COMPLETED` | - | 기사 | 정상 운행 완료 |
| `ORDER_RELEASED` | - | 기사 | 확정 후 내가 포기/방출 |
| `ORDER_CANCELED` | - | 기사/관제탑 | 데스밸리에서 능동 거절 |
| `ORDER_FORCE_CANCELED` | - | 사무실/화주 | 외부에서 강제 취소당함 |

### Phase 1: 타입 정의 통합 (`shared/src/index.ts`)

#### [MODIFY] `OrderStatus` — 단일 통합 라이프사이클 타입

```typescript
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 콜(Order) 통합 라이프사이클 상태 (DB 영구 저장 + 메모리 관리)
// 모든 상태값에 ORDER_ 접두사를 붙여 다른 도메인 상태와 즉각 구분
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
export type OrderStatus =
    // --- [심사 및 결재 단계] ---
    | 'ORDER_PRE_SECURED'          // (패널티 X) 리스트 겉표면 검토 중
    | 'ORDER_SECURED_EVALUATING'   // (패널티 O) 상세화면 진입, 카카오 연산 중

    | 'ORDER_COMPLETED'            // 운행 완료 (정상 종료)
```

### UI 정책: 완료/방출 카드 → 필터링으로 처리

완료(`ORDER_COMPLETED`)되거나 방출(`ORDER_RELEASED`)된 카드는 별도 UI 정책을 하드코딩하지 않고, **기존 `PinnedRoute`의 필터링 로직**에서 해당 상태값을 제외함으로써 자연스럽게 리스트에서 빠지도록 처리합니다. 필요 시 "완료된 콜 보기" 토글 필터를 추가하면 다시 보이게 할 수도 있어, 확장성까지 확보됩니다.

```typescript
// PinnedRoute.tsx 기존 필터 로직에 상태값만 추가하면 끝
const liveRoute = activeRoute.filter(r => 
    r.status !== 'ORDER_COMPLETED' && 
    r.status !== 'ORDER_RELEASED' &&
    r.status !== 'ORDER_CANCELED' &&
    r.status !== 'ORDER_FORCE_CANCELED'
);
```
