### 배차 엔진 Backlog
- [ ] **2번 폰 확정 실패 팝업 자동 닫기**: 1번 폰이 먼저 콜을 확정한 경우, 2번 폰은 인성앱에서 "이미 배차된 콜" 등의 에러 팝업(POPUP_ERROR)을 받게 됨. 이 팝업의 '닫기' 버튼을 자동으로 클릭하여 리스트 화면으로 복귀시키는 로직 필요.
- [ ] **관제탑 상태 분리 (Page vs Hold)**: screenContext(화면 상태)와 isHolding(서버 대기 상태)를 별도 변수로 분리하여 대시보드에 독립적으로 표시. 현재 데스밸리 중 화면 상태가 멈추는 버그 해결 목적.

### SaaS 전환 후 순차적으로 진행할 Backlog (보류 항목)
- [ ] **소켓 텔레메트리 브로드캐스트 유저별 분리**: 현재 `socketHandlers.ts`에서 `io.emit("telemetry-devices", ...)`로 전체 접속자에게 모든 기기 목록을 1초마다 브로드캐스트하고 있음. SaaS 멀티테넌트 완성 시 `io.to(userId).emit(...)` 방식으로 변경하여 기사 본인 소유 기기만 보이도록 분리 필요. (PIN 연동 구현 시 Q4 합의사항)
- [ ] **Android 파서 앱 보안 강화**: 무인 단말기 API 엔드포인트(`POST /api/scrap`)에 대해 `API_KEY` 발급 및 인증 헤더 등 보안 로직 적용 검토 (비인가 오더 폭격 방어).
- [ ] **어드민(Admin) 전용 관제 시스템 고도화**: 전체 오더 스트림을 브로드캐스팅하는 대신, 어드민은 시스템 연결 상태나 에러 통계, 빌링 지표 등 '상태 요약' 대시보드를 구축. 최초 어드민 지정은 환경 변수(`SUPER_ADMIN_EMAIL`) 방식 사용.
- [ ] **개별 기기 페어링 UX 강화 (QR 코드)**: 수동 단축 코드 입력 방식에서 나아가, 안드로이드 앱에 기기 연결용 QR 코드를 띄우고 인식시키는 플로우 구현.
- [ ] **프론트엔드 토큰 저장소 보안 고도화**: JWT 저장을 `localStorage` 방식에서 `HttpOnly Cookie`로 이전하여 웹 XSS 위협 근본 차단 검토 (현재는 편의상 local storage 유지).
