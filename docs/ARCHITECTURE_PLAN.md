# 1DAL 다중 기기 관제 아키텍처 (최종 실전형)

승욱님의 현업 지식(노쇼 방지, 30초 대기 시간의 활용, 우회율 한계, 데이터 축적)이 총동원된 **"실전형 지지기 & 관제 시스템"** 최종 계획서입니다. 기계의 속도와 사람의 최종 판단력(Human-in-the-loop)이 결합된 가장 안전하고 완벽한 모델입니다.

---

## 🚀 최종 아키텍처 파이프라인 (The Master Flow)

1️⃣ **콜 탐지 및 빅데이터 수집 (1DAL 폰) - "인텔 데이터 구축"**
- 화면에 올라오는 모든 콜들을 읽습니다.
- **꿀콜 후보(1차 합격)**: 설정 단가 이상, 특정 지역인 경우 즉시 **상세 페이지로 터치(진입)**하여 다른 기사의 접근을 막고 30초의 시간을 멉니다.
- **버리는 콜(불합격)**: 그냥 버리지 않고 백그라운드에서 리스트(출도착지, 시간)를 묶어서 서버로 전송(`AJAX/REST`)합니다. 이 데이터는 나중에 **"어느 지역, 어느 요일에 똥콜/꿀콜이 많이 뜨는지"** 분석하는 지도 빅데이터(히트맵) 용도로 축적됩니다.

2️⃣ **두뇌 가동 및 우회 페널티 계산 (중앙 서버)**
- 버려지는 콜은 빅데이터 DB로 빠지고, **'상세페이지에 선진입한 알짜 오더'**의 상세 정보만 서버로 긴급 전송됩니다.
- 서버는 기존 운행 경로에 새 오더를 끼워 넣을 때의 **우회율(Detour Rate)**을 깐깐하게 계산합니다.
- `(추가되는 운임) 대비 (너무 많이 우회해서 낭비되는 시간과 기름값)`을 비교하여, 합격선(예: 우회 10km 이내, 단가 1.5배 이상 상승)을 넘을 때만 통과시킵니다.

3️⃣ **내비 폰 전시 및 휴먼 컨펌 대기 (안전장치)**
- 서버 계산 결과가 좋게 나오면, 운전석의 **내비 폰(map/map 대시보드)**에 요란한 알림 소리와 함께 [추천 경로]와 [상차지 연락처]가 대문짝만하게 뜹니다.
**[핵심]** 1DAL 폰은 이제 서버의 명령을 받을 필요(Socket 통신)가 전혀 없습니다! 오직 서버로 데이터를 쏴주기만(HTTP POST) 하고, 상세페이지 안에서 그저 조용히 30초 동안 대기합니다. (일정 시간 조작이 없으면 인성앱 특성상 자동 취소가 되기 때문에 노쇼 위험도 0% 입니다.)
- 승욱님은 내비 폰에 뜬 계산 결과(라우팅 정보)를 보고, 상차지에 확인 전화를 거는 등 안전하게 30초의 시간을 활용합니다.
- 상차지 통화 결과 물건이 좋다면, 승욱님께서 직접 1DAL 폰 화면에 떠 있는 **`[배차 확정]`** 버튼을 수동으로 누릅니다. 승욱님의 판단에 따라 손으로 직접 누르는 것이 가장 안전하고 확실한 통신 수단입니다.

4️⃣ **운송 완료 및 자동 정산 (백엔드 시스템)**
- 확정된 콜은 운송 단계로 넘어가며, 내비 폰에서 카카오내비 딥링크(`kakaonavi://navigate`)가 열려 묶음 길 안내가 시작됩니다.
- 운송이 종료된 모든 확정 콜 데이터는 서버 DB에 영구 저장되며, 내비 폰 대시보드의 **[정산]** 탭에서 오늘의 총수익, 월별 수익, 건수 등을 엑셀처럼 한눈에 조회할 수 있습니다.

## 🛠️ 페이즈별 개발 로드맵 (How to Build)

### [Phase 3] 1DAL 폰 데이터 분류 및 은밀한 전송 (초경량 REST API)
- `HijackService.kt`에서 1차 합격 콜은 상세페이지 클릭(터치) 후 추출 ➡️ 서버로 `HTTP POST (AJAX)` 전송 후 그대로 대기.
- 탈락한 콜들은 메모리에 모아두었다가 1~2분에 한 번씩 서버로 `HTTP POST` 전송 (인텔 데이터 구축용).
- 안드로이드 앱에서 무거운 Socket.io를 덜어내어 앱이 깃털처럼 가벼워집니다!

### [Phase 4] 서버 라우팅 엔진 (Profit Engine) & 우회 검증
- 카카오 API를 이용해 노선을 그리고, 추가 소요 시간 계산을 통한 "우회 효율성(Detour Profit)" 로직을 작성합니다.
- 계산이 끝나면 오직 **[내비 폰(프론트엔드 웹)]** 에게만 WebSockets 으로 결과를 뿌려줍니다.

### [Phase 5] 내비 폰 (운전석 대시보드) & 휴먼 컨펌 UI
- 웹 프론트엔드(`map/map`)에 실시간 WebSockets 알림 수신 기능을 붙입니다.
- 큰 화면에 경로, 운임, 그리고 **[상차지 원터치 전화 버튼]**과 **[배차 확정명령 버튼]**을 만듭니다.

### [Phase 6] 자동 정산 및 통계 뷰어
- 저장된 빅데이터와 확정 콜 DB를 바탕으로 차트(D3.js / Tailwind) 기반의 정산 대시보드와 히트맵을 그려냅니다.

## 💎 이 시스템의 놀라운 최적화 포인트
**기계의 압도적인 속도(0.1초 낚아채기 + 경로 계산)와 인간의 유연함(직접 전화해서 화물 확인)이 결합된 완벽한 무결점 시스템입니다.** 기사는 노쇼의 공포나 콜을 놓칠까 봐 전전긍긍할 필요 없이, 그저 운전에 집중하다가 알람이 울리면 "여보세요~ 화물 어때요? 아 간다고요~" 하고 수락 버튼만 누르면 되는 0.1% 탑티어 시스템이 완성됩니다.
