승욱님의 수익 창출 머신으로 바꿔줄 **일달(1DAL)**의 기술 요소를 정리해 드립니다. 

기존의 웹 아키텍처(Next.js/React)는 데이터 시각화와 관리용으로 유지하되, 실제 인성 앱을 컨트롤할 **안드로이드 네이티브 자동화 엔진**이 핵심 기술 스택으로 추가되어야 합니다.

---

## 🛠️ 일달(1DAL) 기술 스택 및 구성 요소 (Tech Stack)

### 1. 하이재킹 엔진 (Android Native)
인성 앱 위에 상주하며 화면을 읽고 제어하는 핵심 엔진입니다.
* **언어**: Kotlin (안드로이드 최신 표준 및 고성능 비동기 처리)
* **AccessibilityService (접근성 서비스)**: 
    * 인성 앱의 UI 트리(Node)를 분석하여 [배차], [확정], [취소] 버튼의 좌표를 추출.
    * 사용자 대신 버튼을 클릭하는 **Auto-Click** 구현.
* **MediaProjection API**: 실시간 화면 캡처를 통해 OCR 엔진에 데이터 전달.
* **Overlay Window (System Alert Window)**: 인성 앱 화면 위에 수익성 지표(Green/Yellow/Red)를 띄워주는 플로팅 UI.

### 2. 데이터 추출 및 분석 (OCR & Perception)
화면 속 텍스트를 실시간으로 디지털 데이터화합니다.
* **Google ML Kit (Text Recognition)**: 온디바이스(On-device)에서 초고속으로 상/하차지, 요금, 물품 정보를 추출 (오프라인 작동 가능).
* **Regex Engine**: 추출된 텍스트에서 "광주", "성남", "10,000" 등 핵심 키워드와 숫자만 골라내는 정규식 패턴 매칭.

### 3. 수익성 계산 알고리즘 (Profit Engine)
승욱님의 물리적 환경을 반영한 필터링 로직입니다.
* **Cost Calculator**: 
    * **차량 데이터**: BMW 5 GT 연비(13.8km/L), 경유가 실시간 연동.
    * **공정 가치**: 주행 거리당 타이어/오일 감가상각비 계산.
* **Logistics Logic**: 
    * **합짐 동선 분석**: 현재 [확정]된 콜들과 새 콜의 직선/주행 거리 편차 계산.
    * **복귀 우선순위**: 현재 GPS 위치와 경기도 광주 댁 사이의 벡터 방향 일치도 계산.

### 4. 관제 및 설정 (Web/App Hybrid)
기존 승욱님의 강점인 웹 기술을 활용한 관리 영역입니다.
* **Frontend**: Next.js / Tailwind CSS (운전 중 시인성을 극대화한 다크모드 및 고대비 UI).
* **State Management**: TanStack Query (보정 상수 및 당일 수익 데이터 동기화).
* **Storage**: SQLite (기기 내부 로그 저장) & Supabase (전체 주행 데이터 및 수익 통계 클라우드 저장).

## 🏗️ 시스템 아키텍처 (Architecture)

| 레이어 | 주요 기능 | 비고 |
| :--- | :--- | :--- |
| **인식층 (Perception)** | 화면 스캔 -> OCR -> 데이터 구조화 | 0.1초 이내 처리 목표 |
| **판단층 (Intelligence)** | 수익성 계산 -> 합짐 여부 판단 -> 등급 부여 | 5 GT 전용 보정 상수 적용 |
| **실행층 (Action)** | 자동 클릭(탁송/배차) -> 알림 발생 | 3개월 대기 후 2구 확장 대응 |
| **관리층 (Management)** | 주행 로그 기록 -> 수익 통계 -> 설정 변경 | 웹 대시보드 연동 |

## 📝 향후 개발 로드맵 (Roadmap)

1.  **Phase 1 (인식)**: `AccessibilityService`를 활용해 인성 앱의 상세 페이지 텍스트(요금, 목적지)를 로그로 찍어보기.
2.  **Phase 2 (수익)**: 5 GT 연비를 적용한 '건당 순수익' 계산기 모듈 연동.
3.  **Phase 3 (자동화)**: 수익이 기준치 이상일 때 주황색/파란색 버튼 자동 터치 구현.
4.  **Phase 4 (최적화)**: 6개 확정 콜의 위치를 지도에 뿌려주는 최적 순회 경로(TSP) 알고리즘 추가.
