    participant 관제탑 as 🖥️ 관제웹<br/>(브라우저)
    participant 서버 as ☁️ 1DAL 서버<br/>(Node.js)
    participant DB as 💾 SQLite
    participant 카카오 as ☁️ 카카오 API
    participant 앱폰1 as 📱 앱폰1<br/>(단독콜 기기)
    participant 앱폰2 as 📱 앱폰2<br/>(합짐콜 기기)
    participant 인성DB as ☁️ 인성 서버

    Note over 앱폰1: 📍 백그라운드 Telemetry (lat, lng) 획득

    서버-->>앱폰1: 13. [HTTP 폴링] 응답 시 최신 단독콜 필터 주입
    Note over 앱폰1: 안드로이드 자체 메모리에 필터 캐싱 (0.01초 광클 준비)

    Note over 관제탑, 앱폰2: [PHASE 2] 단독콜 선점 (광클) 및 팝업 무인 서핑
    
    인성DB-->>앱폰1: 14. [인성 Socket] 신규 오더 리스트 푸시
    Note over 앱폰1: 1차 리스트 필터 통과 (차종, 거리, 단가)
    앱폰1->>인성DB: 15. [인성 Socket] 해당 오더 클릭 (가로채기!)
    인성DB-->>앱폰1: 16. 상세페이지 진입 및 데이터 노출
    Note over 앱폰1: 2차 텍스트 필터(적요 등) 통과 확인

    Note over 앱폰1: 🔒 isHolding = true 설정 (이후 화면 요동쳐도 상태 락 유지)

    Note over 앱폰1: 🏄‍♂️ 무인 서핑 (적요상세 ➡️ 출발지 ➡️ 도착지) 반복
    앱폰1->>인성DB: 20. [인성 Socket] 각 팝업별 버튼 터치하여 상세 호출
    인성DB-->>앱폰1: 21. 연락처, 세부 주소 등 세부 데이터 노출
    Note over 앱폰1: 화면 텍스트 긁어서 변수에 누적 저장 후 닫기
    
    앱폰1->>서버: 22. [HTTP 폴링] POST /orders/detail (긁어모은 상하차지+적요 전송)
    서버->>관제탑: 23. [Socket] 관제탑 UI에 출발지 가안 및 적요 내용 선출력
    end

    %% =========================================================
    %% [PHASE 3] 카카오 연산 및 관제탑 결재 (기동 방어)

    Note over 관제탑, 앱폰2: [PHASE 3] 좌표 지오코딩, 동선 탐색 및 관제탑 결재

    서버->>카카오: 24. [지오코딩 API] 정제 텍스트로 정확한 좌표 요청

    카카오-->>서버: 27. 폴리라인(도로 궤적), 소요 시간, 거리, 통행료 반환

            서버->>카카오: 30. [내비 재요청] Priority 변경

            서버->>관제탑: 32. 렌더링 갱신
        end

        서버->>관제탑: 28-1. [Socket] 에러 코드 및 얼럿 모달 전송
    end

        서버-->>앱폰1: 35. [HTTP 폴링] 응답으로 Action=Keep 명령
        Note over 앱폰1: 즉각 '닫기' 버튼 터치 후 🔓 isHolding = false (락 해제)
        Note over 앱폰1: 합짐 필터를 덮어쓰고 새로운 먹잇감 감시 돌입
    end
    end

    end
