# 🛣️ 1DAL 스마트 회랑 (동적 합짐) 아키텍처

> 이 문서는 기존 `합짐계획1.md` ~ `합진계획5.md`까지의 논의와 기획을 하나로 통합한 마스터 기획서입니다.
> 현재 파이프라인과 뼈대(UI 다형성, State Machine, 서버 제어)는 100% 구현 완료되었으며, Turf.js를 이용한 실제 지리 연산 엔진부만 구현 대기 중입니다.

---

## 1. 🏗️ 아키텍처 철학 (Architecture Philosophy)

새로 도입되는 "스마트 합짐 모드"는 시스템 성능 하락 없이 유연성을 극대화하는 **동적 노선 기반 필터링**입니다.

1. **Server-Heavy, Client-Light**: 모든 공간 연산(Turf.js)과 경로 추출은 서버 백엔드가 전담합니다. 
2. **분기 없는 다형성 (Polymorphism)**: 클라이언트 프론트엔드는 "합짐인지, 첫짐인지"를 따지지 않고, 오직 서버가 던져준 텍스트 덩어리를 매핑할 뿐입니다.
3. **API 비용 제로화**: 합짐 영역을 찾기 위해 카카오 API 역지오코딩을 반복 호출하지 않고, 로컬의 `merged_map.geojson`을 서버 메모리에 올려 최단 시간 내에 교차점을 찾습니다.

---

## 2. 🔀 동적 UI 매핑 (Polymorphic Overloading)

프론트엔드(`OrderFilterStatus.tsx`) 소스코드에는 어떠한 if문 조건이나 스타일 분기 로직도 들어가지 않습니다. 서버가 주는 아래 데이터를 무조건 순서대로 화면에 뿌립니다.

> `[isSharedMode] | [minFare] | [pickupRadiusKm] | [destinationCity]([destinationRadiusKm]) | [model]`

서버가 상황에 따라 데이터를 주무르면 프론트엔드 UI는 알아서 옷을 갈아입습니다.

**[ 매핑 시나리오 1: 첫짐 세팅 ]**
- 서버 페이로드: `isSharedMode: false, minFare: 4.0, pickupRadiusKm: 30, destinationCity: "광주시", destinationRadiusKm: 10`
- 화면 출력 ➡️ **`첫짐 | 4.0 | 30km | 광주시(10km) | 다마스`**

**[ 매핑 시나리오 2: 합짐 세팅 (광역 10km 버퍼 기준, 현재 반경 0km 진입) ]**
- 서버 페이로드: `isSharedMode: true, minFare: 4.0, pickupRadiusKm: 5, destinationCity: "10km확장 | 영동고속 - 남양로 - 경충대로", destinationRadiusKm: 0`
- 화면 출력 ➡️ **`합짐 | 4.0 | 5km | 10km확장 | 영동고속 - 남양로 - 경충대로(0km) | 다마스`**

## 3. 🔄 코어 배차 파이프라인 (Execution Flow)

우리(서버)와 기사(앱)가 동작하는 흐름은 다음과 같이 완벽히 일치합니다.

| 단계 | 주체 | 동작 핵심 내용 | 상태 |
| :--- | :--- | :--- | :--- |
| **1. 낚시** | 앱 | 빈차 상태에서 `첫짐 필터(수동)` 레이더를 가동하여 콜을 탐색합니다. (`isSharedMode: false`) | 구현 완료 |
| **2. 확정** | 앱/웹 | 기둥콜(본콜)을 잡으면, 웹엔 줌(Zoom) 기능이 제한된 깔끔한 단일 지도 캔버스(`PinnedRoute`)가 그려지고 상/하차지 핀이 꽂힙니다. | 구현 완료 |
| **3. 연산** | 서버 | 카카오 노선을 바탕으로 Turf.js를 가동, `merged_map.geojson`을 스캔하여 경로 주변 읍/면/동 50여 개를 도출하고 `destinationKeywords` 안에 숨겨 담아냅니다. | **미구현** |
| **4. 주입** | 서버 | 다음 `/api/scrap` 하트비트 시, 서버는 방금 만든 JSON(`isSharedMode: true`)을 응답(`dispatchEngineArgs`)으로 몰래 내려줍니다. | 구현 완료 |
| **5. 덮어쓰기** | 앱 | 앱은 이 데이터를 받아 기존 필터를 전부 파괴하고, 서버판 '합짐 필터'로 100% 덮어쓰기 합니다. 이제 앱은 내가 지나갈 회랑(50개의 동네)에서 뜨는 꿀콜만 스크래핑합니다. | 구현 완료 |

## 4. 🛡️ 심화 방어 로직 (Deep Dive Defenses - TODO)

합짐 효율을 극대화하기 위해, 시스템에 향후 도입할 심화 연산 방어 로직입니다. (현재 `todo.md`에 백로그로 등록됨)

### 4.1 톨게이트 우회 방지 (Highway Detour Penalty)
직선상으로 가까워도 고속도로면 국도를 빙 돌아야 하는 물류의 치명적 딜레마를 서버 연산으로 방어합니다.
* 카카오 API `guides`에서 해당 구간이 `고속국도`/`도시고속도로` 판정을 받으면 Turf.js 합짐 반경 연산을 0km로 제재하여 고속도로 톨게이트를 나갔다 들어오며 헤매는 불상사를 원천 차단합니다.

### 4.2 위치 기반 오토 스위칭 (GPS Auto-Strict)
차량이 화물을 싣고 움직이기 시작하여 **실주행** 속도가 파악되면, 서버는 어떠한 조작 없이도 강제로 `destinationRadiusKm`을 `0`(정밀 노선)으로 깎은 필터를 앱에 주입시킵니다.
* 이를 통해 주행 중 노선 이탈 확률을 소멸시키고 가는 길목에 위치한 콜만 정확하게 잡아냅니다.
