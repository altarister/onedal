# 🏢 1DAL SaaS 전환 (Multi-Tenant) 및 인증 아키텍처

> 이 문서는 기존 `auth1.md`, `auth2.md`, `auth3.md`의 기획안을 하나로 통합한 최종 인증/권한 아키텍처 문서입니다. 
> 현재 1DAL 시스템의 데이터베이스(SQLite), 구글 로그인(JWT), 소켓 분리(Socket.io Rooms)에 **100% 구현 완료**되어 작동 중입니다.

---

## 1. 🌐 아키텍처 대전환: Multi-Tenant (다중 사용자 격리)
기존의 "1대의 스마트폰 + 중앙 관제" 형태를 완벽히 탈피하여, 다수의 기사(User)들이 각자의 스마트폰과 필터 설정을 가지고 독립적으로 동작하는 **SaaS(Software as a Service)** 형태로 구축되었습니다.

### 1-1. JWT + Socket.io 룸(Room) 기반 통신
서버는 수많은 안드로이드 폰과 관제웹 접속을 동시에 안전하게 감당합니다.
- **JWT (JSON Web Token)**: 로그인 시 발급되는 토큰으로 REST API 및 소켓 핸드셰이크를 검증합니다.
- **Socket Isolation**: 각 기사의 소켓 연결은 `socket.join(userId)`를 통해 프라이빗 룸에 격리됩니다.
- **완벽한 보안 보장**: 기사 A의 폰에서 올라온 콜 데이터와 상태 갱신은 오직 `io.to('기사A_UID').emit(...)` 형태로만 뿌려져 다른 기사의 화면과 절대 섞이지 않습니다.

---

## 2. 🗄️ 다중 사용자 데이터베이스 스키마 (SQLite)
서버 전역(Global) 변수로 존재하던 모든 설정이 DB화되어 기사별(Per-User)로 할당되었습니다.

### 2-1. `users` & `user_devices` (회원 및 기기 매핑)
기사님 1명이 다수의 단말기(안드로이드 파서 폰)를 사용할 수 있도록 설계되었습니다.
- 대시보드에서 "기기 페어링"을 통해 폰의 `deviceId`를 본인 계정에 귀속시킵니다.
- 이후 해당 폰에서 올라오는 스크랩 데이터는 전부 해당 유저의 운행 일지로 기록됩니다.

### 2-2. `user_filters` (기사별 오더 사냥 필터)
과거 전역 메모리에 있던 `activeFilterConfig`가 DB화 되었습니다.
기사마다 목적지(`destination_city`), 수입 기준(`min_fare`), 회랑 반경(`corridor_radius_km`) 등 사냥 룰이 1:1로 격리 저장됩니다.

### 2-3. `user_settings` (기사별 라우팅 및 연비 개인화)
동일한 콜을 잡아도 카카오 API가 내려주는 "추천 경로"와 "순수익"은 기사의 차량 제원에 따라 달라집니다.
- **연비 개인화**: 유종(`car_fuel`), 리터당 유가(`fuel_price`), 연비(`fuel_efficiency`), 하이패스 장착 여부(`car_hipass`)
- **라우팅 개인화**: 선호 경로(`default_priority`), 톨게이트 회피 여부(`avoid_toll`) 등

---

## 3. 🛡️ 권한 매트릭스 (ADMIN vs USER)
단순 필터링 변경과 핵심 배차 제어에 대한 권한을 엄격하게 나누었습니다 (`role` 컬럼).

| 기능 영역 | ADMIN (플랫폼 관리자) | USER (일반 기사) |
| :--- | :--- | :--- |
| **필터/오더 제어** | 개별 기사의 필터 및 배차에 원칙상 개입 불가 | **본인의 필터 제어 및 콜 KEEP/CANCEL** |
| **기기 관리** | 시스템 전체 연결 기기 상태/부하 모니터링 | **본인 소유 투폰/쓰리폰 페어링 및 해제** |
| **라우팅 / 수익** | 전체 API 호출량 및 비용/에러 모니터링 | **본인 차량(연비)에 특화된 경로/정산 확인** |
| **운행 일지(Intel)**| `intel` 전체 빅데이터 통계 및 열람 | **본인의 기기가 스크랩한 내역/일지 열람** |
| **사용자 관리** | 신규 가입자 승인, 이용 정지, 권한 변경 | 본인 프로필 관리 |

> 💡 **Tip:** 
> 현재 `auth` 라우터와 `authMiddleware.ts`에 JWT 발급 및 Role 가드가 적용되어 있으며, 어드민 기능 확장에 즉시 대응할 수 있는 상태입니다.
