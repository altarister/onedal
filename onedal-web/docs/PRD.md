# 1DAL-WEB PRD (Product Requirements Document)

## 📌 제품 개요
`onedal-web`은 안드로이드 스캐너(`onedal-app`)가 잡아낸 콜 데이터를 실시간으로 수신하여, 운전자의 대시보드(스마트폰 크롬 브라우저)에 시각화해주는 **콜 지휘 통제소(Cockpit)** 입니다.

## 👥 핵심 유저 시나리오 (User Journey)
1. **관제 시작**: 운전자(승욱님)가 주력 스마트폰(관제 폰)으로 크롬 브라우저를 열고 `onedal-web` 주소에 접속합니다.
2. **무중단 대기**: 브라우저는 화면 꺼짐 방지(Wake Lock)를 가동한 채 거치대에서 계속 켜져 있습니다. 서버와의 실시간 소켓 연결은 우측 상단 🟢 아이콘으로 확인됩니다.
3. **오더 수신 및 피드백**:
   - 스캐너가 콜을 잡으면, 화면 상단에 거대한 [콜 카드]가 팝업되며 즉시 시끄러운 경고음("띵띵!") 시스템 사운드가 울립니다.
   - 운전 중 시야를 뺏기지 않도록 **출발지, 도착지, 요금** 3가지 항목만 화면의 80%를 덮을 만큼 거대하게 다크모드로 렌더링 됩니다.
4. **휴먼 인 더 루프 (배차 검토)**:
   - 운전자는 카드를 슬쩍 보고, 하단의 `[🗺️ 카카오내비]` 버튼을 눌러 현재 경로 대비 우회율을 눈으로 빠르게 스캔합니다.
   - 동선이 맞는 합짐 콜이라면 `[📞 상차지 전화]` 버튼을 눌러 화주/기사에게 전화를 걸어 물건 크기를 확인하고 배차를 최종 수락합니다.

## ✨ 상세 기능 및 화면 명세서 (UI/UX)

### 1. 메인 대시보드 (`/`)
- **[헤더 영역]**: 오늘 날짜, 현재 누적 수익금, **Socket.io 실시간 연결 상태(🟢/🔴)**.
- **[오더 카드 타일]**:
  - 최신 콜이 맨 위에 쌓임 (Stack).
  - 배경색: 운전 중 눈부심 방지를 위한 고대비 다크 테마(Tailwind `bg-slate-800` 등).
  - 데이터: 강렬한 폰트 크기로 `pickup ➡️ dropoff (fare원)`. 원문 데이터(`rawText`)는 클릭 시 펼쳐지는 형태로 보조 수단으로 활용.
- **[원터치 액션 버튼]**:
  - `전화걸기` 버튼: HTML `<a href="tel:010...">` 을 활용하여 브라우저에서 안드로이드 다이얼러로 즉각 화면 전환.
  - `길안내` 버튼: `<a href="kakaonavi://navigate?name=도착지">` 딥링크를 활용해 내비게이션 자동 세팅.

### 2. 정산 및 콜 히스토리 (`/settlement`)
- 오늘 스캐너가 낚아채서 화면에 올려준 모든 콜의 로그가 노출됩니다.
- 각 콜 우측에 `[완료/포기]` 체크박스가 있습니다.
- 완료 처리된 콜들의 운임만 모두 합산되어 상단에 **"오늘 총 순수익: 150,000원"** 식으로 전시됩니다.

## ⚠️ 엣지 케이스 및 예외 처리
- **터널 진입 등 5G 끊김 (소켓 단절)**: 소켓 연결이 끊기면 즉시 상단 아이콘이 🔴로 변하고, 브라우저가 자동으로 지속적인 재접속(Auto-Reconnect)을 시도합니다. 연결이 복구되면 누락되었던 콜을 서버에서 HTTP GET으로 한 번에 끌어옵니다.
- **오디오 권한 이슈**: 크롬 정책상 '사용자의 첫 터치(User Interaction)'가 없으면 알림음(Web Audio) 재생이 차단됩니다. 따라서 첫 화면 진입 시 "소리 켜기" 버튼을 누르도록 강제하는 초기 진입 팝업을 설계합니다.
