# 1DAL-APP PRD (Product Requirements Document)

## 📌 제품 개요
`onedal-app`은 기사의 개입 없이 인성앱(타겟 물류 앱)의 화면을 실시간으로 읽어내 수익성이 높은 콜을 **0.1초 내외로 선점(자동 터치)**하는 안드로이드 전용 매크로 및 데이터 수집 엔진입니다.

## 👥 핵심 유저 시나리오 (User Journey)
1. **앱 실행 및 권한 승인**: 사용자가 앱을 켜면 '접근성 권한', '다른 앱 위에 표시 권한', '배터리 최적화 예외'를 요구받고 승인합니다.
2. **서비스 시작**: 메인 화면에서 [엔진 가동 버튼]을 누르면 백그라운드 서비스가 켜지고(알림바 고정), 화면 구석에 작은 **상태 오버레이(초록색 🔴/🟢)**가 뜹니다.
3. **인성앱 대기**: 사용자는 인성앱을 켜두고 화면을 봅니다. (혹은 글로브박스에 거치합니다).
4. **자동 사냥 (Auto-Hunt)**:
   - 인성앱 콜 리스트에 새 오더가 올라오면, 눈에 보이기도 전에 1DAL 엔진이 텍스트를 파싱합니다.
   - **수익성 합격**: 시스템이 강제로 해당 콜의 \[상세\] 영역 좌표를 터치합니다.
   - **수익성 불합격**: 터치하지 않고, 해당 콜의 데이터를 서버로 발송해 빅데이터로 저장합니다.

## ✨ 상세 기능 요구사항 (Features)

### 1. 화면 스캐너 (Screen Scanner)
- **트리거**: 인성앱 안에서 화면이 스크롤되거나 새로운 리스트가 갱신될 때마다 즉각 발동.
- **데이터 파싱 방식**: 정규식을 맹신하지 않고, `"요금"`, `"@"`, `"/"` 등의 앵커를 찾는 **Best-Effort 휴리스틱(Fuzzy) 파서**를 가동.
- **데이터 추출 항목**:
  - `pickup` (상차지, `@` 심볼 기준 추출)
  - `dropoff` (하차지, `/` 심볼 기준 조립 추출)
  - `fare` (운임 요금, "요금" 텍스트 근처 숫자 추출)
  - `rawText` (스캐너가 긁어온 원본 텍스트 전체. 휴리스틱 실패 대비용)

### 2. 수익성 판별 엔진 (Profit Filter)
- 파싱된 텍스트를 기반으로 점수를 매깁니다.
- **합격 기준 (차후 고도화 예정)**: 
  - 특정 단가표 이상 (예: 5만원 이상)
  - 특정 선호 목적지 포함 (예: "성남", "광주", "강남")
- 한 번 스캔했던(이미 서버로 넘긴) 콜의 ID나 텍스트 해시를 기억하여 **중복 처리를 방지**합니다.

### 3. 강제 터치 액션 (Auto-Click)
- 합격 판정 시, 파싱 로직에서 찾아낸 해당 리스트 아이템의 **화면 X, Y 좌표(BoundingBox)** 중앙값을 즉시 탭(Tap) 합니다.
- 터치 완료 후, 상세 페이지에 진입하면 엔진은 **30초간 대기 모드(휴식)**로 전환됩니다. (기사님이 관제 폰에서 배차를 최종 수락/거절할 시간을 벌어줌)

### 4. 플로팅 UI (Overlay Window)
- 화면 우측 상단에 50x50 픽셀 크기의 미니 위젯 제공.
- 🟢 **초록색**: 스캔 가동 중 (이상 없음)
- 🟡 **노란색**: 꿀콜 발견, 상세페이지 진입 완료 (대기 중)
- 🔴 **빨간색**: 시스템 오류 또는 인성앱 이탈 상태

### 5. 무음 데이터 전송 (Stealth HTTP POST)
- 모든 스캔 결과(합격/불합격)를 서버(`onedal-web`)로 조용히 보냅니다.
- 기기 성능을 갉아먹지 않도록 일정 개수(예: 10개)가 모이거나, 5초 주기로 묶어서(Bulk) 전송합니다.

## ⚠️ 예외 및 엣지 케이스 (Edge Cases)
- **인성앱이 백그라운드로 내려갈 때**: 스캐너를 일시 중지하여 불필요한 배터리 소모를 막습니다.
- **OS가 앱을 강제 종료(Kill)할 때**: 안드로이드 Foreground Service 알림을 띄워 OS가 절대 죽이지 못하게 방어합니다.
- **화면 구성(UI) 업데이트**: 인성앱 디자인이 바뀌어 노드 ID가 달라져도 작동할 수 있게, ID 의존 방식보다는 Regex(글자 패턴) 및 구조 뎁스 기반으로 우회 파싱합니다.
